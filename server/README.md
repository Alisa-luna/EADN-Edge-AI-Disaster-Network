**下面将详细介绍当前服务器具有的功能**

### 一、数据接收与存储

| 功能 | 实现位置 | 说明 |
|------|----------|------|
| **MQTT 订阅** | `MQTTHandler` 类 | 订阅 `earthquake/data`、`earthquake/alert`、`earthquake/heartbeat` 三个主题 |
| **节点事件解析** | `_handle_eq_event()` | 解析网关上传的 `eq_event` 类型 JSON，提取 τc、S-P、方位角等参数 |
| **融合事件接收** | `_on_message()` | 接收网关在 4G 模式下自己融合后上传的 `fused_event` |
| **心跳监控** | `_on_message()` | 接收网关定期发送的心跳包，监控在线状态 |
| **SQLite 存储** | `EarthquakeDatabase` 类 | 保存所有事件 (`events` 表)、节点报告 (`node_reports` 表) 和小时基线 (`hourly_baselines` 表) |

---

### 二、多节点震源定位

| 功能 | 实现位置 | 说明 |
|------|----------|------|
| **多节点网格搜索定位** | `locate_epicenter()` | 当 2 个以上节点同时报警时，在节点平均坐标周围 ±0.5° 范围内搜索最佳震中 |
| **单节点方位角推算** | `_single_node_epicenter()` | 只有 1 个节点时，用 P 波方位角 + 震中距推算震中位置 |
| **无方位角回退** | `_trigger_event()` | 节点没有方位角时，以节点自身坐标作为震中参考 |
| **震级融合** | `_trigger_event()` | 各节点 s_peak 按距离修正后加权平均，得到融合震级 |

---

### 三、单节点可信度评估

| 功能 | 实现位置 | 说明 |
|------|----------|------|
| **AI/AE 一致性** | `compute_single_node_score()` | 将 AI 置信度和 AE 异常分数融合为一致性指标 |
| **τc 物理一致性** | 同一函数内 | 验证 τc 与 s_peak 是否符合经验公式 `log10(τc) ≈ 0.5×log10(s_peak) - 1.5` |
| **S-P 距离一致性** | 同一函数内 | 验证 S-P 走时与震中距是否符合 8 km/s 波速关系 |
| **时间一致性** | 同一函数内 | 对比节点时间戳与服务器 NTP 时间 |

---

### 四、CENC 地震目录接入

| 功能 | 实现位置 | 说明 |
|------|----------|------|
| **REST API 拉取** | `CENCFetcherV2` 类 | 通过 `/list` 和 `/lastid` 接口获取国家地震科学数据中心最新地震目录 |
| **增量检测** | `get_last_id()` | 通过 `/lastid` 接口判断是否有新事件，避免重复拉取 |
| **事件详情获取** | `fetch_event_detail()` | 通过 `/event/{id}` 获取包含台站数据的详细信息 |
| **USGS 备用数据源** | `_fallback_usgs()` | CENC API 不可用时，自动切换至 USGS 全球地震目录 |
| **首次全量展示** | `cenc_monitor()` | 首次启动时拉取所有历史事件并打印详细信息 |
| **常规轮询** | 同一函数内 | 之后每 30~60 秒检查一次最新事件 |

---

### 五、烈度计算与评估

| 功能 | 实现位置 | 说明 |
|------|----------|------|
| **震级经验公式** | `IntensityCalculator.calculate()` | 用中国东部衰减模型 `I = 3.5 + 1.2M - 3.0×log(R) - 0.002R` 估算本地烈度 |
| **仪器烈度衰减** | `IntensityCalculatorV2.calculate()` | 优先使用 CENC 提供的仪器烈度进行距离衰减计算 |
| **深度修正** | 同一函数内 | 中源地震 (>70km) 烈度降低 1.5 度，深源地震 (>300km) 降低 2.5 度 |
| **距离衰减** | 同一函数内 | 震中距越大，烈度衰减越多 |

---

### 六、告警与通知

| 功能 | 实现位置 | 说明 |
|------|----------|------|
| **MQTT 警报发布** | `EventManager._trigger_event()` / `AlertDispatcher.dispatch()` | 融合事件或外部地震事件触发后，发布到 `earthquake/alert` |
| **邮件告警** | `send_email_alert()` | 通过 QQ 邮箱 SMTP 发送 HTML 格式告警邮件 |
| **钉钉告警** | `AlertDispatcher._send_dingtalk_v2()` | 通过钉钉机器人 Webhook 发送 Markdown 格式告警 |
| **烈度分布图生成** | `generate_shakemap_html()` | 生成高德地图热力图，标注震中和节点烈度 |
| **国标烈度颜色** | `AlertDispatcherV2.dispatch()` | 按 GB/T 38226-2019 标准使用烈度颜色和罗马数字 |
| **告警去重** | `alerted_events` 集合 | 同一事件只告警一次 |

---

### 七、数据可视化

| 功能 | 实现位置 | 说明 |
|------|----------|------|
| **烈度分布 HTML 地图** | `generate_shakemap_html()` | 高德地图 + 热力图，标记震中位置和各节点烈度 |
| **震中标记** | 同一函数内 | 红色脉冲动画标记推定震中 |
| **节点烈度标记** | 同一函数内 | 各节点按烈度显示不同颜色的标签 |
| **事件信息卡片** | 同一函数内 | 地图右上角显示时间、震级、置信度等信息 |

---

### 八、事件管理与关联

| 功能 | 实现位置 | 说明 |
|------|----------|------|
| **事件窗口关联** | `EventManager.add_report()` | 30 秒内到达的节点报告归为同一事件 |
| **事件触发判定** | `_trigger_event()` | 2 个以上节点报警时自动触发融合定位 |
| **事件数据库写入** | `EarthquakeDatabase` 类 | 将融合后的事件信息和各节点报告写入 SQLite |
| **事件历史查询** | `get_event_count()` | 获取数据库中历史事件总数 |

---

### 九、网络管理

| 功能 | 实现位置 | 说明 |
|------|----------|------|
| **MQTT 自动重连** | `MQTTHandler.connect()` | 最多重试 5 次，间隔 5 秒 |
| **SSL/TLS 加密** | `MQTTHandler.__init__()` | 支持 CA 证书验证，证书不可用时自动降级 |
| **多 Topic 订阅** | `_on_connect()` | 同时订阅 data、alert、heartbeat 三个主题 |

---

### 十、日志与监控

| 功能 | 实现位置 | 说明 |
|------|----------|------|
| **文件日志** | `logging` 配置 | 所有日志同时输出到 `eq_server.log` 文件和控制台 |
| **事件详情日志** | `_handle_eq_event()` | 记录每个节点报告的完整参数 |
| **CENC 拉取日志** | `cenc_monitor()` | 记录每次拉取的事件数量和详情 |
| **告警日志** | 各告警函数 | 记录 MQTT、邮件、钉钉的发送状态 |

---

## 功能分类总结

| 大类 | 核心功能 |
|------|----------|
| **数据接收** | MQTT 订阅、节点事件解析、融合事件接收、心跳监控 |
| **数据存储** | SQLite 事件表、节点报告表、基线表 |
| **震源定位** | 多节点网格搜索、单节点方位角推算、震级融合 |
| **可信度评估** | AI/AE 一致性、τc 物理一致性、S-P 距离一致性 |
| **CENC 接入** | REST API 拉取、增量检测、USGS 备用、首次全量展示 |
| **烈度计算** | 震级经验公式、仪器烈度衰减、深度修正 |
| **告警通知** | MQTT 警报、QQ 邮件、钉钉机器人、烈度分布图 |
| **可视化** | 高德热力图、震中动画、节点烈度标签 |
| **事件管理** | 时间窗口关联、融合触发、数据库写入 |
| **网络管理** | MQTT 重连、SSL 加密、多主题订阅 |
